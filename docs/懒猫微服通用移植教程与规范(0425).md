# 懒猫微服（Lazycat MicroServer）通用项目移植教程与规范

> **版本**：v2.0 ｜ **最后更新**：2026-04-25
>
> 本文档是 AI Agent 执行懒猫微服移植任务的**唯一权威指引**。全文按执行优先级从高到低排列，Agent 必须从上到下逐步执行，不得跳步。

---

## ⚡ 第零章 红线规则（最高优先级 — 任何时候都不得违反）

> **本章内容在所有章节之上，发生冲突时以本章为准。**

### 0.1 绝对禁止清单

1. **不得在 `buildscript` 脚本中调用 `lzc-cli project build`** — 会导致无限递归。
2. **不得将非 `registry.lazycat.cloud` 的镜像用于正式发布。**
3. **不得写入 `/lzcapp/pkg/content/`** — 该路径只读。所有可写数据必须在 `/lzcapp/var/` 或 `/lzcapp/cache/`。
4. **不得自行捏造 `author` 字段** — 必须从上游仓库获取（见 0.2）。
5. **不得遗漏 `locales.ja`** — `locales` 必须包含 `zh`、`zh_CN`、`en`、`ja` 四个语言。
6. **不得遗漏 `package.yml`** — 且 `package`、`version`、`author` 必须与 `lzc-manifest.yml` 完全一致。
7. **不得在无需要时接管 80/443 端口。**
8. **不得随意挂载非 `/lzcapp` 开头的系统路径。**
9. **OIDC 登录仅在原项目有登录功能时才实现** — 不是每次移植都需要（见第五章）。

### 0.2 Agent 必须执行的信息采集

在生成任何配置文件之前，Agent **必须**先执行以下命令获取项目元信息：

```bash
# 在项目根目录执行
git remote -v                    # 获取仓库地址 → 用于确定 author、homepage
cat package.json 2>/dev/null     # Node.js 项目
cat pyproject.toml 2>/dev/null   # Python 项目
cat pom.xml 2>/dev/null          # Java 项目
cat Cargo.toml 2>/dev/null       # Rust 项目
cat go.mod 2>/dev/null           # Go 项目
```

从上述输出中提取：
- **`author`**：仓库所有者 / `package.json` 的 `author` / `pyproject.toml` 的 `authors` / `pom.xml` 的 `developers`
- **`name`**（项目名）：仓库名 / 包名
- **`homepage`**：仓库 URL
- **`license`**：LICENSE 文件或元数据中的 license 字段

**若无法自动获取，必须向用户询问，不得猜测或填写占位符。**

## 第一章 移植前评估（Gate 0 — 全部通过才可继续）

Agent 收到移植需求后，必须按顺序执行以下 5 项检查，输出评估报告后才能进入第二章。

### 1.1 检查应用商店是否已上架

搜索懒猫官方应用商店。若已上架且版本满足需求，告知用户无需移植。

### 1.2 拦截纯前端项目

**判断标准** — 满足以下全部条件视为纯前端：仅含静态文件（HTML/CSS/JS/图片），无任何服务端进程，所有 API 均指向外部第三方。

若判定为纯前端，Agent **必须暂停**并输出：

```
⚠️ 检测到该项目为纯前端项目（无后端/服务端组件）。
纯前端项目无需移植到懒猫微服即可直接部署在任意静态托管服务上。
本次移植流程已暂停，如需继续请明确特批。
```

**未获特批不得继续。**

以下**不属于**纯前端：Next.js SSR / Nuxt.js / SvelteKit 等前后端一体应用；需要本地 API 代理的 SPA；前端 + Python/Node.js 后端的组合架构。

### 1.3 判断是否为 Web 项目 + 中间件依赖

**Web 适配性**：只有通过 HTTP 提供 UI 或 API 的项目才适合移植。CLI / 桌面应用需要评估改造难度（简单 / 中等 / 困难）。

**中间件红线**：
- ✅ 允许：SQLite、H2、LevelDB/BoltDB、MongoDB（同容器后台进程）
- ❌ 禁止：PostgreSQL、MySQL、Redis、RabbitMQ/Kafka、Elasticsearch、Selenium/Playwright

命中禁止列表且无法替换为轻量方案的，告知用户不建议移植。

### 1.4 评估平台兼容性

懒猫微服运行 Linux amd64。确认编译产物架构正确，不支持的客户端平台需在 manifest 中用 `unsupported_platforms` 声明。

### 1.5 询问打包方式（v1 或 v2）

Agent **必须**在评估阶段向用户确认采用哪种打包方式：

| 对比项 | v1（`application` + `upstreams`） | v2（`services` 多容器） |
|--------|-----------------------------------|-------------------------|
| 架构 | 单容器，业务进程通过 `backend_launch_command` 拉起 | 多容器，每个 service 独立镜像 |
| 路由 | `upstreams` → `http://127.0.0.1:PORT` | `routes` → `http://service.appid.lzcapp:PORT` |
| 启动入口 | 需编写 `run.sh` | 由镜像自身 entrypoint 启动 |
| 最低系统版本 | 无特殊要求 | **lzcos ≥ v1.5.0**（需在 manifest 中声明 `min_os_version: "1.5.0"`） |
| 适用场景 | 轻量应用、源码构建、嵌入运行时 | 已有 Docker 镜像、多服务编排 |

Agent 向用户输出：

```
请确认本次移植采用的打包方式：
1. v1（单容器 + upstreams）— 无系统版本要求
2. v2（多容器 services）— 需要 lzcos ≥ v1.5.0
```

**用户回复后才可继续，不得自行决定。**

### 1.6 输出评估报告

包含：项目名称与类型、商店上架情况、是否纯前端、Web 适配性、中间件依赖分析、打包方式确认、预计踩坑点、最终结论（建议移植 / 有条件移植 / 不建议移植 / 纯前端暂停等待特批）。

结论为"建议移植"或用户接受风险后，进入第二章。

## 第二章 文件结构与核心配置

### 2.1 必须文件清单

```
项目根目录/
├── lzc-manifest.yml          # 运行配置
├── package.yml               # 静态元数据（package/version/author 必须与 manifest 一致）
├── lzc-build.yml             # 构建入口
├── lzc-icon.png              # 应用图标（1:1 PNG，≤200KB）

```

### 2.2 `lzc-manifest.yml` — 规范模板

#### v1 模板（单容器 + upstreams）

```yaml
package: cloud.lazycat.app.your-app-name    # 反域名格式，全局唯一
version: 1.0.0                               # 语义化版本 X.Y.Z
name: 应用显示名称
description: 一句话描述
author: 上游仓库原作者                       # ⚠️ 必须从 git remote / 元数据获取
license: MIT
homepage: https://github.com/原作者/项目名

locales:
  zh:
    name: "中文名"
    description: "中文描述"
  zh_CN:
    name: "中文名"
    description: "中文描述"
  en:
    name: "English Name"
    description: "English description"
  ja:
    name: "日本語名"
    description: "日本語の説明"

application:
  image: registry.lazycat.cloud/xxx/debian:tag  # 留空则用默认 Alpine 3.21
  subdomain: yourapp
  background_task: false

  upstreams:
    - location: /
      backend: http://127.0.0.1:8080
      disable_trim_location: true
      backend_launch_command: /bin/sh /lzcapp/pkg/content/lzc/run.sh
```

#### v2 模板（多容器 services）

```yaml
package: cloud.lazycat.app.your-app-name
version: 1.0.0
name: 应用显示名称
description: 一句话描述
author: 上游仓库原作者                       # ⚠️ 同 v1，必须从上游获取
license: MIT
homepage: https://github.com/原作者/项目名
min_os_version: "1.5.0"                      # ⚠️ v2 必须声明

locales:
  zh:
    name: "中文名"
    description: "中文描述"
  zh_CN:
    name: "中文名"
    description: "中文描述"
  en:
    name: "English Name"
    description: "English description"
  ja:
    name: "日本語名"
    description: "日本語の説明"

application:
  subdomain: yourapp
  routes:
    - /=http://main.cloud.lazycat.app.your-app-name.lzcapp:8080

services:
  main:
    image: registry.lazycat.cloud/xxx/your-image:tag
    binds:
      - /lzcapp/var/data:/data
    environment:
      - ENV_KEY=value
```

### 2.3 关键字段速查

| 字段 | 规则 |
|------|------|
| `package` | `cloud.lazycat.app.` 前缀 + 项目名，全局唯一 |
| `version` | 严格语义化 X.Y.Z |
| `author` | **必须**来自上游仓库（第零章 0.2 获取），严禁捏造 |
| `locales` | 必须含 `zh`、`zh_CN`、`en`、`ja`，每个含 `name` + `description` |
| `application.image` | 正式发布必须 `registry.lazycat.cloud` 开头 |
| `upstreams[].backend`（v1） | `http://127.0.0.1:PORT` |
| `routes`（v2） | `http://servicename.appid.lzcapp:PORT` |
| `min_os_version`（v2） | 必须 `"1.5.0"` 以上 |

### 2.5 `lzc-build.yml`

```yaml

manifest: ./lzc-manifest.yml

pkgout: ./

```

### 2.6 `lzc/run.sh`（v1 必须，v2 可选）

```bash
#!/bin/sh
set -e

APP_DIR="/lzcapp/pkg/content"
VAR_DIR="/lzcapp/var/your-app"
CACHE_DIR="/lzcapp/cache/your-app"

mkdir -p "$VAR_DIR" "$CACHE_DIR"

# 首次拷贝默认配置
if [ ! -f "$VAR_DIR/config.yml" ] && [ -f "$APP_DIR/default-config.yml" ]; then
    cp "$APP_DIR/default-config.yml" "$VAR_DIR/config.yml"
fi

# 启动主进程（必须用 exec）
cd "$VAR_DIR"
exec python3 "$APP_DIR/server.py" --port 8080 --data-dir "$VAR_DIR"
```

**硬性要求**：`chmod +x`、关键依赖做存在性检查、用 `exec` 启动主进程、端口与 `upstreams[].backend` 一致、不写 `/lzcapp/pkg/content/`。

## 第四章 语言栈踩坑指南

### 4.1 Java / Spring Boot

核心模型：fat jar + 嵌入 Linux amd64 JRE。

关键踩坑点：JRE 必须是 Linux amd64（校验 `file` 输出为 `ELF 64-bit LSB`）；前后端一体项目先 `pnpm build` 再 `bootJar`；JRE 下载慢时用 `LZC_JRE_URL` 切换 BellSoft 直连源。

### 4.2 Go

核心模型：交叉编译 Linux amd64 静态二进制。

关键踩坑点：`GOOS=linux GOARCH=amd64 CGO_ENABLED=0`；`-ldflags '-s -w'` 减体积。

### 4.3 Next.js / Node.js

核心模型：构建产物 + Node 基础镜像（优先使用 `registry.lazycat.cloud/node:*`）。

关键踩坑点：`output: 'standalone'` 减体积；`NEXT_PUBLIC_*` 构建时内联，运行时动态配置用 API route。

### 4.4 Python

核心模型：源码 + 预构建 venv。

**最关键的坑 — 健康检查竞争**：容器启动时 `pip install` 耗时过长 → 健康检查超时 → 容器被杀 → 死循环。**解法：把 venv 和依赖在打包阶段预构建好，容器启动时直接复制，不执行 `pip install`。**

其他踩坑点：PEP 668 限制必须用 venv；Alpine `apk` vs Debian `apt-get` 兼容；venv 放 `/lzcapp/cache/`；升级后需同步代码。

### 4.5 纯前端 SPA

> ⚠️ 需用户特批（见 1.2）。

可用 `file://` 协议直接托管静态文件，或用轻量 Python HTTP 服务器。

## 第五章 OIDC 单点登录（仅在原项目有登录功能时接入）

> **⚠️ 前提条件：OIDC 接入仅适用于原项目本身就有登录功能的情况。如果原项目没有用户系统/登录页面，不需要也不应该实现 OIDC 接入。**

### 5.1 原理

懒猫微服 v1.3.5+ 提供统一 OIDC 支持。开发者只需做两件事：

1. 在 manifest 中声明 `oidc_redirect_path`（OIDC 回调地址）
2. 把系统自动生成的 OIDC 环境变量传给业务进程

系统检测到 `oidc_redirect_path` 后会自动生成 `LAZYCAT_AUTH_OIDC_*` 环境变量。没有这个字段，环境变量不会被注入。

### 5.2 manifest 配置

#### v1 写法（application + upstreams）

```yaml
application:
  subdomain: yourapp
  oidc_redirect_path: /auth/oidc.callback
  environment:
    - OIDC_CLIENT_ID=${LAZYCAT_AUTH_OIDC_CLIENT_ID}
    - OIDC_CLIENT_SECRET=${LAZYCAT_AUTH_OIDC_CLIENT_SECRET}
    - OIDC_AUTH_URI=${LAZYCAT_AUTH_OIDC_AUTH_URI}
    - OIDC_TOKEN_URI=${LAZYCAT_AUTH_OIDC_TOKEN_URI}
    - OIDC_USERINFO_URI=${LAZYCAT_AUTH_OIDC_USERINFO_URI}
  upstreams:
    - location: /
      backend: http://127.0.0.1:8080
      disable_trim_location: true
      backend_launch_command: /bin/sh /lzcapp/pkg/content/lzc/run.sh
```

#### v2 写法（services）

```yaml
application:
  subdomain: yourapp
  oidc_redirect_path: /auth/oidc.callback
  routes:
    - /=http://main.cloud.lazycat.app.yourapp.lzcapp:8080

services:
  main:
    image: registry.lazycat.cloud/xxx/your-image:tag
    environment:
      - OIDC_CLIENT_ID=${LAZYCAT_AUTH_OIDC_CLIENT_ID}
      - OIDC_CLIENT_SECRET=${LAZYCAT_AUTH_OIDC_CLIENT_SECRET}
      - OIDC_AUTH_URI=${LAZYCAT_AUTH_OIDC_AUTH_URI}
      - OIDC_TOKEN_URI=${LAZYCAT_AUTH_OIDC_TOKEN_URI}
      - OIDC_USERINFO_URI=${LAZYCAT_AUTH_OIDC_USERINFO_URI}
```

### 5.3 如何确定 `oidc_redirect_path`

1. 查阅应用官方文档中的 OIDC/OAuth2 配置说明
2. 如果文档没有，先随便填一个值（如 `/auth/callback`），启动后浏览器报错页面会显示实际路径
3. 用实际路径替换后重新打包

### 5.4 可用的环境变量

| 变量名 | 说明 |
|--------|------|
| `LAZYCAT_AUTH_OIDC_CLIENT_ID` | OAuth Client ID（必填） |
| `LAZYCAT_AUTH_OIDC_CLIENT_SECRET` | OAuth Client Secret（必填） |
| `LAZYCAT_AUTH_OIDC_ISSUER_URI` | OIDC Issuer URL（部分应用仅需此项即可自动发现其他端点） |
| `LAZYCAT_AUTH_OIDC_AUTH_URI` | 授权端点 |
| `LAZYCAT_AUTH_OIDC_TOKEN_URI` | Token 端点 |
| `LAZYCAT_AUTH_OIDC_USERINFO_URI` | UserInfo 端点 |

部分应用只需要 `CLIENT_ID` + `CLIENT_SECRET` + `ISSUER_URI`，其余端点会自动探测。部分应用需要显式填写每个端点。查阅应用文档确定需要哪些。

### 5.5 OIDC Issuer 信息

访问 `https://<微服名>.heiyu.space/sys/oauth/.well-known/openid-configuration` 可获取完整的 issuer 信息，包括所有端点地址。端点地址格式为 `https://$LAZYCAT_BOXDOMAIN/$endpoint_path`。

### 5.6 与本地登录共存

如果应用需要同时支持本地账号密码登录和懒猫 OIDC 登录，必须加上：

```yaml
application:
  public_path:
    - /
```

这样请求会先进入应用，由应用自己决定展示本地登录表单还是跳转 OIDC。否则懒猫入口层会拦截登录，用户无法看到应用自己的登录页。

### 5.7 实战示例：Outline

根据 Outline 官方文档，需要设置 `OIDC_CLIENT_ID`、`OIDC_CLIENT_SECRET`、`OIDC_AUTH_URI`、`OIDC_TOKEN_URI`、`OIDC_USERINFO_URI`：

```yaml
application:
  subdomain: outline
  oidc_redirect_path: /auth/oidc.callback    # 从报错信息中获得
  routes:
    - /=http://outline.cloud.lazycat.app.outline.lzcapp:3000

services:
  outline:
    image: registry.lazycat.cloud/tx1ee/outlinewiki/outline:fb0e2ef4f32f3601
    environment:
      - OIDC_CLIENT_ID=${LAZYCAT_AUTH_OIDC_CLIENT_ID}
      - OIDC_CLIENT_SECRET=${LAZYCAT_AUTH_OIDC_CLIENT_SECRET}
      - OIDC_AUTH_URI=${LAZYCAT_AUTH_OIDC_AUTH_URI}
      - OIDC_TOKEN_URI=${LAZYCAT_AUTH_OIDC_TOKEN_URI}
      - OIDC_USERINFO_URI=${LAZYCAT_AUTH_OIDC_USERINFO_URI}
```

### 5.8 注意事项

- 不要用 `inject` 模拟自动登录，OIDC 场景走标准授权码流程
- 报 `Invalid character ' '` 错误说明 `scope` 参数未正确 URI 编码
- 本地账号映射建议：用户名优先 `preferred_username`，权限组从 `groups` 读取（含 `ADMIN` 可映射为管理员）

## 第六章 路由配置

### 6.1 v1 标准路由（upstreams）

```yaml
application:
  upstreams:
    - location: /
      backend: http://127.0.0.1:8080
      disable_trim_location: true
      backend_launch_command: /bin/sh /lzcapp/pkg/content/lzc/run.sh
```

### 6.2 v2 标准路由（routes）

```yaml
application:
  routes:
    - /=http://main.cloud.lazycat.app.your-app-name.lzcapp:8080
```

注意：`routes` 默认会去掉路径前缀。需保留前缀时改用 `upstreams` + `disable_trim_location: true`（lzcos v1.3.9+）。

### 6.3 高级路由选项（upstreams）

| 选项 | 用途 |
|------|------|
| `disable_trim_location: true` | 保留 URL 路径前缀（推荐始终开启） |
| `use_backend_host: true` | Host 头使用 backend 中的 host |
| `remove_this_request_headers: [Origin, Referer]` | 清除请求头，解决跨域 |
| `disable_backend_ssl_verify: true` | 跳过上游 SSL 验证 |

### 6.4 四层转发（ingress）

暴露非 HTTP 端口（如 SSH）。注意：无鉴权保护，严禁接管 80/443。

```yaml
application:
  ingress:
    - protocol: tcp
      port: 22
      description: "SSH 访问"
```

## 第七章 打包、安装与验证

### 7.1 打包

```bash
lzc-cli project build
# 或指定输出：
lzc-cli project build -o myapp.lpk
```

### 7.2 安装

```bash
lzc-cli app install ./cloud.lazycat.app.myapp-v1.0.0.lpk --apk n
```

### 7.3 验证

```bash
lzc-cli app status cloud.lazycat.app.myapp
lzc-cli docker ps
lzc-cli docker logs --tail 300 <container-name>
```

### 7.4 发布前验收

```bash
# 1) 文件存在性
ls -la lzc-manifest.yml package.yml lzc-build.yml lzc-icon.png lzc/build-package.sh lzc/run.sh

# 2) 图标（PNG、≤200KB、正方形）
file lzc-icon.png && ls -la lzc-icon.png

# 3) 脚本语法
sh -n lzc/build-package.sh && sh -n lzc/run.sh

# 4) package / version / author 一致性
grep "^package:" lzc-manifest.yml package.yml
grep "^version:" lzc-manifest.yml package.yml
grep "^author:" lzc-manifest.yml package.yml

# 5) locales.ja 存在
grep -A2 "ja:" lzc-manifest.yml

# 6) 镜像来源
grep "image:" lzc-manifest.yml | grep -v "registry.lazycat.cloud" && echo "WARNING: 非官方镜像!"

# 7) v2 项目检查 min_os_version
grep "min_os_version:" lzc-manifest.yml

# 8) 打包
lzc-cli project build

# 9) 产物
ls -lah *.lpk
```

## 第八章 常见踩坑汇总

| 症状 | 原因 | 解法 |
|------|------|------|
| `image must from registry.lazycat.cloud` | 镜像非官方源 | `lzc-cli appstore copy-image` 拷贝后写回 manifest |
| `run.sh` 启动失败 | 路径不对 | 用 `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` 自适应 |
| `python3/ffmpeg` 缺失 | 基础镜像未预装 | run.sh 中检查并兼容 `apk` / `apt-get` 安装 |
| `Read-only file system` | 写了 `/lzcapp/pkg/content/` | 改到 `/lzcapp/var/` |
| 升级后代码不生效 | 工作目录缓存了旧代码 | run.sh 每次启动从 `/lzcapp/pkg/content/` 同步 |
| `Permission denied` | 非 root 用户 + 受限目录 | 确保 root 运行或 `chmod`/`chown` |
| 健康检查死循环重启 | 启动阶段耗时太长 | **把依赖安装前移到打包阶段**（根本解法），或调大 `health_check.start_period`（治标） |
| `app status` 查不到 | 包标识符不一致 | `package` 字段与安装命令一致 |

## 附录 A：移植 Checklist

```
□ Gate 0: 商店未上架
□ Gate 0: 非纯前端（或已获特批）
□ Gate 0: Web 项目或改造可行
□ Gate 0: 中间件未命中红线
□ Gate 0: 平台兼容性通过
□ Gate 0: 已确认打包方式（v1/v2）
□ 元信息: author 已从 git remote / 元数据获取
□ 文件: lzc-manifest.yml 符合规范
□ 文件: locales 含 zh / zh_CN / en / ja
□ 文件: package.yml 存在且 package/version/author 一致
□ 文件: lzc-build.yml 完整
□ 文件: lzc-icon.png（1:1 PNG ≤200KB）
□ 文件: build-package.sh 语法正确
□ 文件: run.sh 语法正确且可执行（v1）
□ 镜像: application.image 来自 registry.lazycat.cloud
□ v2专项: min_os_version ≥ "1.5.0"
□ OIDC: 仅在原项目有登录功能时接入
□ 构建: lzc-cli project build 成功
□ 测试: app install + status + 日志 + Web UI 正常
```
